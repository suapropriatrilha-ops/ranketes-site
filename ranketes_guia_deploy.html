Passo 1 — Domínio
REGISTRO.BR
1
Acesse registro.br — é o órgão oficial de domínios .com.br no Brasil. Clique em "Registre seu domínio".
registro.br
REGISTRO.BR — NIC.br
Registre seu domínio .br
ranketes.com.br
Buscar
2
Digite ranketes.com.br na busca. Se estiver disponível, vai aparecer em verde. Clique em "Registrar".
💡
Se ranketes.com.br já estiver registrado por outra pessoa, tente ranketes.app ou getranektes.com.br. Mas tente o .com.br primeiro.
3
Você vai precisar de um CPF ou CNPJ para registrar. Preencha seus dados e finalize o pagamento. O custo é de aproximadamente R$40 por ano.
⚠️
Importante: guarde bem o login e senha do registro.br. Você vai precisar deles no Passo 4 para apontar o domínio para a Vercel.
4
Após o pagamento, o domínio fica ativo em até 24h. Na prática costuma ser menos de 1h. Você receberá um e-mail de confirmação.

✓

Passo 2 — Arquivos
GITHUB
1
O GitHub é onde vamos guardar os arquivos do site. Pense nele como uma pasta online compartilhável — a Vercel vai buscar os arquivos de lá automaticamente. Acesse github.com e crie uma conta gratuita.
2
Após fazer login, clique no botão verde "New" (ou "+ New repository") para criar um repositório.
github.com/new
Create a new repository
Repository name *
ranketes-site
Public
Private
Create repository
3
Preencha assim: Repository name: ranketes-site · Visibilidade: Public · Clique em Create repository.
4
Agora vamos fazer o upload dos arquivos HTML. Na página do repositório, clique em "uploading an existing file" (ou arraste os arquivos diretamente).
📁
Faça o upload de todos os arquivos HTML do Ranketes de uma vez: ranketes_landing.html, ranketes_login.html, ranketes_onboarding.html, ranketes_publico.html, ranketes_card.html, ranketes_professor_jailson.html e os painéis. Depois renomeie o ranketes_landing.html para index.html — este será a página inicial.
5
Após o upload, role a página para baixo e clique em "Commit changes" para salvar. Pronto — seus arquivos estão no GitHub!

Passo 3 — Hospedagem
VERCEL — DEPLOY
1
Acesse vercel.com e clique em "Start Deploying". Faça login com sua conta do GitHub — assim a Vercel já acessa seus repositórios automaticamente.

2
A Vercel vai listar seus repositórios. Clique em "Import" ao lado de ranketes-site.
3
Na tela de configuração, não mexa em nada. Apenas clique no botão azul "Deploy". A Vercel detecta automaticamente que são arquivos HTML simples.
vercel.com/deploy

Framework PresetOther

4
A Vercel vai processar em 30-60 segundos. Quando terminar, vai mostrar uma tela de sucesso com fogos de artifício 🎉 e um link provisório tipo ranketes-site.vercel.app. Clique para ver seu site no ar!
🎉
Neste momento o site já está online e acessível para qualquer pessoa pelo link da Vercel. O próximo passo só troca esse link pelo seu domínio ranketes.com.br.
5
Bônus automático: a cada vez que você atualizar um arquivo no GitHub (subir uma versão nova do HTML), a Vercel vai republicar o site automaticamente em segundos. Sem precisar fazer nada.

Passo 4 — DNS
CONECTAR O DOMÍNIO
1
No painel da Vercel, abra seu projeto e vá em Settings → Domains. Digite ranketes.com.br e clique em Add.
vercel.com/ranketes/settings/domains
Domains
ranketes.com.br
Add
⚠️ Configure DNS
Add the following records to your DNS provider
2
A Vercel vai mostrar os registros DNS que você precisa adicionar. Anote os valores — vão ser parecidos com estes:
TipoNomeValor
A@76.76.21.21
CNAMEwwwcname.vercel-dns.com
⚠️
Use os valores que a Vercel mostrar para você, não os da tabela acima — podem ser diferentes. Copie exatamente o que aparecer na tela.
3
Agora volte ao registro.br, faça login, abra o domínio ranketes.com.br e vá em "DNS" ou "Configurar DNS".
4
Adicione os dois registros que a Vercel indicou (tipo A e CNAME). Salve. O processo de propagação leva de 5 minutos a 48 horas — na maioria das vezes é menos de 1 hora.
🔍
Para verificar se o domínio já está apontando corretamente, acesse dnschecker.org e busque por ranketes.com.br. Quando aparecer o IP da Vercel nos resultados, está funcionando.
5
Quando a propagação terminar, a Vercel vai gerar automaticamente um certificado HTTPS gratuito para seu site. Isso significa que vai aparecer o cadeado 🔒 no navegador — essencial para segurança e confiança dos usuários.

✓

✓
Adicionei os registros no registro.br
✓
ranketes.com.br abre o site com 🔒 HTTPS

Passo 5 — Banco de dados
SUPABASE
1
O Supabase é o banco de dados do Ranketes — é onde ficam guardados atletas, partidas, rankings, professores etc. Acesse supabase.com e crie uma conta gratuita (pode entrar com o GitHub).
🆓
O plano gratuito do Supabase suporta até 50.000 usuários ativos por mês e 500MB de banco de dados. Mais do que suficiente para o MVP do Ranketes.
2
Clique em "New Project". Configure assim:
supabase.com/dashboard/new
Create a new project
Project name
ranketes
Database Password
••••••••••••••
Region
South America (São Paulo)
Create new project
3
Aguarde ~2 minutos para o projeto ser criado. Depois vá em Table Editor (menu lateral) e crie as tabelas principais do Ranketes. Clique em "New table" para cada uma:
TabelaO que guardaCampos principais
usuariosTodos os usuáriosid, email, nome, tipo, criado_em
arenasClubes e arenasid, nome, cidade, estado, plano
atletasPerfil esportivoid, usuario_id, arena_id, pontos, categoria
partidasRegistro de jogosid, vencedor_id, perdedor_id, placar, data
professoresPerfil do professorid, usuario_id, bio, preco_avulsa, arena_id
torneiosTorneios e chavesid, arena_id, nome, formato, status
4
Vá em Authentication → Providers e ative o login por Email. Isso vai permitir que atletas e professores criem conta com e-mail e senha.
🔑
O Supabase já vem com sistema de login, recuperação de senha e tokens de segurança prontos. Você não precisa programar isso do zero — só ativar.
5
Vá em Settings → API e anote a Project URL e a anon public key. Você vai precisar dessas informações quando integrar o backend com o site.
// Exemplo das credenciais que você vai encontrar em Settings → API
const supabaseUrl = 'https://xxxxxxxxxxx.supabase.co'

// Guarde esses valores — você vai usá-los no próximo passo
